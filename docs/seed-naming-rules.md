## 1. 引言

如今自动化影视工具越来越多（如 `Plex`, `Emby`, `Jellyfin`, `Radarr`, `Sonarr`），作为影视剧库管理者，有必要知晓一些音视频编码格式以及常见命名规则。事实上，大多数影视种子名称里面已经包含了该影视文件的一些必要信息，一个例子如下：

```
Interstellar.2014.1080p.BluRay.DDP5.1.x265.10bit-GalaxyRG265
```

---

## 2. 电影基本信息

**`Interstellar.2014`** 代表电影名字（星际穿越）和上映年份。

这部分数据通常被 `Plex` / `Emby` / `Jellyfin` / `Radarr` 等软件用来识别匹配到唯一的电影。现实中电影名字是允许重复的，但是同一年里面上映的电影很少有重名的，因此电影发行名称和年份这个信息基本可以匹配到唯一电影。

## 3. 分辨率

分辨率是决定视频画面清晰度的核心指标，它表示画面由多少个像素点组成。文件名中的分辨率标签能让你一眼就判断出视频的清晰度等级。例子中的 **`1080p`** 就属于分辨率信息。

> ### 核心概念： "p" 与 "i" 的区别
> 
> * **p (Progressive Scan - 逐行扫描)**：这是现代显示技术的主流。每一帧图像都是完整地被扫描和显示的。我们看到的 `1080p`, `720p` 都属于这种，画面稳定无闪烁，观感更好。**绝大多数的电影和剧集资源都是 "p" 格式。**
> * **i (Interlaced Scan - 隔行扫描)**：一种较老的技术，主要用于传统电视广播。它将一帧图像分成奇数行和偶数行两个场，交错显示。这能节省带宽，但高速运动的画面可能会产生梳状条纹。常见的标识是 `1080i`，多见于HDTV录制的体育赛事或新闻节目。
> 
> **简单来说：在同等数字下，`p` 的画质体验优于 `i`。**

`1080p` 代表有1080行像素点。一些地方使用电影载体来指代分辨率，例如DVD，不同载体对应分辨率见后。常见的分辨率有：

| 级别 | 常见标识 | 俗称/别名 | 像素尺寸 (宽x高) | 描述与应用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **超高清 Ultra HD** | `8K` / `4320p` / `UHD` | 8K | 7680x4320 | **目前的画质巅峰**。像素数量是4K的4倍。资源极其稀少，主要用于顶级演示片、IMAX片源或少数高端流媒体内容。文件体积极为庞大。 |
| **超高清 Ultra HD** | `4k` / `2160p` / `UHD` | 4K | 4096x2160 或 3840x2160 | **当前高质量视频的主流标准**。UHD蓝光、高端流媒体平台（Netflix、Disney+）的顶级画质。需要强大的硬件解码能力和支持4K的显示设备。 |
| **全高清 Quad HD** | `2K` / `1440p` / `QHD` | 2K (非严格) | 2560x1440 (或底线2048x1080) | 4倍于HD。应用广泛，是清晰度和文件大小之间的绝佳平衡点。 |
| **全高清 Full HD** | `1080p` /`1080i` / `FHD` | 全高清 | 1920x1080 | **应用最广泛的标准**。几乎所有蓝光碟、WEB-DL、HDTV资源都有此分辨率，是绝大多数用户的收藏首选。 |
| **高清 HD** | `720p` / `HD` | 高清 | 1280x720 | **高清的入门标准**。常见于早期蓝光、HDTV、流媒体平台和动漫番剧。在移动设备或小屏幕上观看效果依然出色。 |
| **标准清晰度 SD** | `576p` / `576i` / `PAL` | 标清 | 1024x576 或 720x576 | 主要来源于欧洲、澳大利亚和中国等地区的DVD或标清电视信号。 |
| **标准清晰度 SD** | `480p` / `480i` / `NTSC` | 标清 | 720x480 或 640x480 | 主要来源于美国、日本等地区的DVD或标清电视信号。是 `DVDRip` 资源最常见的分辨率。 |
| **更低分辨率** | `360p` / `240p` / `144p` | - | 640x360 / 426x240 / 192x144 | 极少出现在正式发布中，通常用于早期网络视频、预览或极差网络环境下的流媒体。 |

## 4. 片源来源

**`BluRay`** 代表片源来自于蓝光原盘，这是判断画质好坏最关键的标签之一。常见来源如下（质量从高到低排序）：

* **`REMUX` / `Blu-ray` / `BluRay` / `BD`**: 蓝光原盘，质量最好。无损地从蓝光光碟中提取视频和音频数据，封装到MKV等格式中，无画质和音质损失。适合收藏，但文件体积巨大（通常几十GB）。
* **`BDRip`**: 对蓝光原盘进行二次编码压制，采用 `H.264` 或 `H.265` 等编码以减小文件体积。画质略低于原盘，但性价比极高，是最常见的资源类型。
* **`WEB-DL`**: 从官方在线流媒体平台（如 Netflix, Amazon）直接下载的视频源，未经二次编码。画质非常好，有时甚至优于BDRip，是目前越来越流行的片源。
* **`WEBRip`**: 通过对流媒体播放画面进行**录屏**获得，画质通常不如 `WEB-DL`。
* **`HDTV`**: 从高清电视频道录制的节目。通常会带有电视台台标，画质受信号质量影响。
* **`HDTVrip` / `HDrip`**: 对 `HDTV` 源进行二次编码压制。
* **`HR-HDTV`**: 半高清，分辨率为1080p的一半左右（如1024x576），是早期人人影视推广的一种标准。
* **`HD-DVD`**: 已被蓝光淘汰的历史格式，目前流传的资源多为2009年以前发行。
* **`DVDRip`**: 从最终版的DVD转制而来，分辨率较低（`480p`或`576p`）。
* **`DVDScr`**: 预览版DVD，画质稍差于 `DVDRip`，可能带有版权滚动信息。常见于颁奖季送审样片。
* **`R5`**: 特指俄罗斯5区的DVD，通常发行比欧美早，但视频质量和音效较差，有时会合成枪版音轨。
* **`TC` (Telecine)**: 胶片版，使用电视电影机从胶片直接数字拷贝，画质尚可但可能偏暗，音质通常较差。
* **`TS` (Telesync)**: 枪版，使用外置音源（如耳机孔）录音，音质比 `CAM` 稍好。画面在投影室录制，可能比 `CAM` 稳定。
* **`CAM` (Camera)**: 枪版，用摄像机在影院直接盗录，画面晃动，声音嘈杂，质量最差。

## 5. 视频编码

例子中的 **`x265`** 代表视频编码格式。不同的编码技术决定了视频的压缩率和画质。

* **`MPEG-2`**: DVD和早期HDTV的标准，技术较老，压缩效率低。
* **`VC-1`**: 由微软主导，与H.264同期竞争，压缩比略低于H.264，但解码消耗资源较少。
* **`MPEG4` (XviD/DivX)**: `H.264` 出现之前的网络视频霸主，压缩率高，成像清晰。
* **`H.264` / `AVC` (x264编码器)**: **曾经的王者，应用最广泛**。在1080p时代是绝对的主力，兼容性极强。`x264`是其最著名的开源编码器。
* **`VP8`**: Google主导的开放格式，与`WebM`容器和`Vorbis`音频搭配使用。
* **`VP9`**: `VP8`的继任者，与`H.265`竞争，是YouTube的主力编码格式。
* **`PRORES`**: 苹果公司开发的高品质有损压缩格式，主要用于后期制作，可支持高达8K。
* **`H.265` / `HEVC` (x265编码器)**: **现在的主流，4K/8K时代的核心**。压缩效率比`H.264`高约30-50%，意味着在相同画质下，文件体积可以小很多。`x265`是其著名的开源编码器。
* **`AV1`**: **未来的标准，开放且免版税**。由谷歌、苹果、微软等巨头组成的开放媒体联盟开发，压缩效率比`H.265`更高，但对硬件要求极高。

## 6. 视频色深

例子中的 **`10bit`** 指的是视频色深，常见的还有 `8bit`。

色深代表像素点色彩空间使用几个bit来表示。我们知道计算机中所有颜色都是由红绿蓝（RGB）混合而成，每一种颜色称为一个通道。
* **`8bit`**: 最常见的标准，每个颜色通道用8位表示，总共能显示约1670万种颜色（24位真彩色）。
* **`10bit`**: 每个通道用10位表示，能显示约10.7亿种颜色，色彩过渡更平滑，能有效减少色阶断层。

**注意**：观看 `10bit` 视频需要显示设备支持，否则观感与 `8bit` 无异。此外，`10bit` 视频由于压缩算法更优，有时文件体积可能比同等画质的 `8bit` 视频更小。

---

## 7. 色彩动态范围

色彩动态范围 (`HDR` vs `SDR`) 决定了画面最亮和最暗部分的细节表现力。HDR能提供更广的亮度范围和更丰富的色彩，使画面更接近人眼在真实世界所见的景象。

* **`HDR10`**: 开源、免费的基础HDR标准，应用最广泛。采用**静态元数据**，对整个影片使用一组固定的亮度信息。
* **`Dolby Vision` / `DV` (杜比视界)**: 由杜比实验室推出，采用**动态元数据**，可以逐帧或逐场景优化亮度、对比度和色彩，效果惊艳。需要硬件和软件认证，并支付授权费用。
* **`HDR10+` / `HDR10PLUS`**: `HDR10`的升级版，同样采用**动态元数据**，效果媲美杜比视界，但开源免费。
* **`HLG` (Hybrid Log Gamma)**: 由BBC和NHK联合开发，主要用于广播电视。特点是无需元数据，能同时兼容HDR和SDR显示设备。
* **`HDR Vivid`**: 由中国超高清视频产业联盟主导推出的HDR标准，支持动态元数据，已成为中国广电行业的标准。

## 8. 音频格式

例子中的 **`DDP5.1`** 是声音格式。它包含两部分：
* **`DDP`**: 音频编码格式（Dolby Digital Plus）。
* **`5.1`**: 声道形式（5个方向声道 + 1个重低音声道）。同理，`7.1`是8声道，`2.0`是立体双声道。

#### 常见音频编码 (有损/无损)
* **有损压缩**: `MP3`, `AAC`, `WMA`, `OGG/VORBIS`, `OPUS`
* **无损压缩**: `FLAC`, `APE`, `WAV`, `ATRAC`

#### 4K蓝光中常见的高级音频格式
* **`Dolby Digital` / `DD` / `AC3`**: 从DVD时代就普及的环绕声格式，蓝光基础标准之一。
* **`DTS`**: `AC3`的主要竞争对手，通常码率更高，理论上音质更好。
* **`Dolby Digital+` / `DD+` / `DDP` / `EAC3`**: `DD`的增强版，码率更高，支持多达7.1声道。
* **`DTS-HD HR` (High Resolution Audio)**: `DTS`的高分辨率版本，码率更高。
* **`PCM`**: 未经压缩的录音棚母版复制品，音质最纯净，但占用空间巨大。
* **`Dolby TrueHD`**: **无损压缩**格式，解码后与录音棚母版完全一致，占用空间小于PCM。
* **`DTS-HD MA` (Master Audio)**: **无损压缩**格式，与`Dolby TrueHD`类似，是DTS的顶级格式。
* **`Dolby Atmos` (杜比全景声)**: 基于**声音对象**的3D环绕声技术，能营造出声音在三维空间中移动的真实感，是目前家庭影院的顶级标准之一。
* **`DTS:X`**: 与杜比全景声竞争的基于**声音对象**的沉浸式音频技术，对音箱布局要求更灵活。

## 9. 发布组信息

例子中的 **`GalaxyRG265`** 代表发布组名称。

不同的发布组压制视频有不同的风格和质量标准。`NOGRP` 表示来源不明或非特定组织发布，类似于「佚名」。

---

## 10. 发行商/电视台信息

有的电视剧集片源还带有发行商和电视台信息，这通常暗示了片源的原始质量和字幕支持情况。

* **`ATVP`**: Apple TV+ 原创剧集。
* **`NF`**: Netflix 原创剧集。
* **`AMZN`**: Amazon Prime Video 原创剧集。
* **`DSNP`**: Disney+ 原创剧集。

以上流媒体发行商的片源普遍对多语言字幕支持较好，推荐下载。

---

## 11. 帧率

电影最常见的帧率是 **24 帧/秒** (电影产业标准) 和 **30 帧/秒**。随着高刷新率显示设备的流行，一些压制组会制作 `60FPS` 或 `120FPS` 的视频，码率相对更高，动态画面更流畅。

---

## 12. 封装容器/后缀

封装容器（或称“后缀”）决定了文件如何将视频流、音频流和字幕流打包在一起。

* **`mkv` (Matroska)**: **后起之秀，高清时代的主流**。功能最强大的封装格式，可容纳多种不同编码的视频、音频及多条字幕流，是高清/4K资源的首选。
* **`mp4`**: **手机、平板的新宠儿**。兼容性极佳，几乎所有设备都支持，是网络流媒体和移动视频的主流格式。
* **`ts`**: 高清格式，优点是对硬件要求不高，缺点是文件体积巨大。通常是蓝光原盘或电视录制的原始数据流。
* **`avi`**: **廉颇老矣**。诞生早，兼容性好，但技术老旧，不支持现代功能，已被`mkv`基本取代。
* **`mov`**: 苹果公司开发的格式，常用于高清预告片和专业视频制作。
* **`rmvb`**: 曾在中国非常流行，但技术早已过时，不适合高清视频压制。
* **`flv` / `webm`**: 主要用于网络流媒体，如直播和网页视频。

## 13. 音轨语言信息

* **`Dual.Audio` / `Dual-Audio`**: 双语音轨。
* **`Tri-Audio`**: 三语音轨。
* **`Multi`**: 多语种音轨。
* **`CHINESE` / `RUSSIAN` / `KOREAN` 等**: 直接标记出片源所包含的非英语音轨。
* **`DUBBED`**: 表示片源包含译制配音音轨，而非演员原声。

## 14. 字幕信息

### 14.1. 内挂/内封字幕
字幕被封装在视频文件中，但可以由播放器自由选择、切换、隐藏或调整样式。常见的格式有 `SRT`、`ASS`、`PGS` 等。

### 14.2. 外挂字幕
独立的字幕文件（如 `.srt`, `.ass`），与视频文件放在同一目录下，播放时由播放器自动加载。
* **`.srt`**: 最基本、最流行的纯文本字幕。
* **`.ssa` / `.ass`**: 高级字幕，支持丰富的自定义样式和特效，是动漫字幕组的首选。
* **`.sup` / `.sub`+`.idx`**: 图形字幕，从蓝光或DVD中提取，无法编辑文本内容。

**常见命名后缀：**
* **语言**: `.zh.srt` (中文), `.en.srt` (英文)。
* **简繁**: `.chs.srt` (简体), `.cht.srt` (繁体)。
* **听障人士字幕**: `.hi` / `.cc` / `.sdh`。

### 14.3. 硬字幕
又称**内嵌字幕**，字幕与视频画面被压制在一起，像水印一样，**无法关闭或修改**。
* **`SUBBED`**: 标签表示资源带有硬字幕。
* **`UNSUBBED`**: 与`SUBBED`相对，表示没有硬字幕。

---

## 15. 剧集/季信息

* **整季打包 (Season Pack)**: 文件名中包含 `Season 01` / `S01` / `S1` 等字眼，表示第一季完整内容。
    * `Westworld.S01.Season.1.Complete.HDTV.720p.x265.AAC.E-Subs`
* **单集/多集**: 文件名中包含 `S01E02` (第一季第二集) 或 `S01E01-E03` (第一季1至3集)。
    * `Loki.S01E01.1080p.DSNP.WEBRip.DDP5.1.Atmos.x264-NOGRP`
* **动漫/无季概念**: 可能只有剧集编号，如 `[06]`。
    * `Na.Nare.Hana.Nare.[06][1080p][WebRip]`
* **综艺/日期编号**: 少数综艺节目可能以播出日期作为剧集编号。
    * `The.Genius.2013.20150912.1080p.x264`

---

## 16. 种子大小信息

一些发布组（如 `GalaxyRG265`）会在种子标题中包含大致的文件大小信息，例如 `800M` 表示文件大小在800MB左右。

## 17. 其它杂项

* **`PROPER`**: 完美版。表明此版本优于之前其他小组发布的有问题的版本。
* **`REPACK`**: 修正版。发布组对自己之前发布的有问题的版本进行修正后重新发布。
* **`Theatrical.Cut`**: 剧场版（影院上映版本）。
* **`Director.Cut` / `DC`**: 导演剪辑版。
* **`Extended`**: 加长版。
* **`UNRATE`**: 未删节版。
* **`LIMITED`**: 限制上映版（通常指在少数影院上映的艺术电影等）。
* **`IMAX`**: 巨幕版，通常画面比例更高，能显示更多内容。
* **`Open.Matte`**: 满屏版，与宽屏版相比，通常上下黑边更少，画面内容更多。
* **`REMASTERED` / `4K`**: 重制版/4K修复版，指对老电影进行画质修复和重新灌录。
* **`CEE` / `EUR` / `US` / `HK`**: 指蓝光光盘的发行区域（中欧/欧洲/美国/香港），不同区域的版本可能在音轨和字幕配置上有所不同。`CEE`版通常因包含中文字幕而受国内用户欢迎。
* **`WITH.EXTRAS`**: 带花絮。
* **`WITH.OST`**: 带原声音乐CD。
* **`WS`**: 宽屏幕 (Widescreen)。
* **`FS`**: 全屏幕 (Fullscreen)。
* **`NUKED`**: 被废弃的版本，因画质太差或其他严重问题。
