## 1. 引言
如今自动化影视工具越来越多，作为影视剧库管理者有必要知晓一些音视频编码格式以及常见命名规则，事实上大多数影视种子名称里面已经包含了该影视文件的一些必要信息，一个例子如下：
`Interstellar.2014.1080p.BluRay.DDP5.1.x265.10bit-GalaxyRG265`

---
## 2. 电影基本信息
`Interstellar.2014` 电影名字（星际穿越）和上映年份。这部分数据通常被 Plex/Emby/Jellyfin/Radarr 等软件用来识别匹配到唯一的电影；现实中电影名字是允许重复的，但是同一年里面上映的电影很少有重名的，因此电影发行名称和年份这个信息基本可以匹配到唯一电影。

## 3. 分辨率
例子里面为1080p，事实上因显示设备比例不同，可能还有许多其他分辨率格式，大致规律为例如1080p代表有1080行像素点。在许多地方我们也能见到诸如1080i这样的分辨率格式，1080p与1080i的区别在于前者为逐行扫描，后者为隔行扫描模式。一些地方使用电影载体来指代分辨率，例如DVD，不同载体对应分辨率见后；常见的分辨率有：

| 级别 | 常见标识 | 俗称/别名 | 像素尺寸 (宽x高) | 描述与应用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **超高清 UHD** | `8K` / `4320p` | 8K UHD | 7680x4320 | **目前的画质巅峰**。资源极其稀少，主要用于顶级演示片、IMAX片源或少数高端流媒体。文件体积极为庞大。 |
| **超高清 UHD** | `4K` / `2160p` / `UHD` | 4K UHD | **DCI 4K**: 4096x2160 <br> **UHD**: 3840x2160 | **当前高质量视频的主流标准**。`DCI 4K`是数字影院标准，`UHD`是消费级电视/蓝光标准，文件名中的`4K`或`2160p`通常指后者。 |
| **准高清** | `2K` / `1440p` / `QHD` | 2.5K / Quad HD | 2560x1440 | 介于FHD和UHD之间，在影视资源中不常见，主要用于PC显示器和部分高端手机屏幕。 |
| **全高清 FHD** | `1080p` | FullHD | 1920x1080 | **应用最广泛的标准**。几乎所有蓝光、WEB-DL、HDTV资源都有此分辨率，是绝大多数用户的收藏首选。 |
| **全高清 (广播)** | `1080i` | - | 1920x1080 | 隔行扫描，动态画质**劣于**`1080p`。多见于HDTV录制的体育赛事、演唱会或新闻节目。 |
| **高清 HD** | `720p` | HD | 1280x720 | **高清的入门标准**。常见于早期蓝光、HDTV、流媒体和动漫番剧。 |
| **标清 SD (PAL)** | `576p` / `576i` | PAL / SDTV | 1024x576 或 720x576 | 主要来源于欧洲、澳大利亚和中国等地区的DVD或标清电视信号。`576p`是逐行，`576i`是隔行。 |
| **标清 SD (NTSC)** | `480p` / `480i` | NTSC / SDTV | 720x480 或 854x480 | 主要来源于美、日等地区的DVD或标清电视信号。`480p`是逐行，`480i`是隔行。 |
| **低分辨率** | `360p` / `240p` / `144p` | VCD画质 | 各种更低尺寸 | 极少出现在正式发布中，通常用于视频预览或早期网络视频。 |
| **特殊标识 (负面)** | `Upscaled` | 拉伸/伪高清 | - | **质量欺诈**。指由低分辨率通过算法拉伸至高分辨率的资源，其实际清晰度远低于原生高分辨率。**应尽量避免**。 |
| **特殊标识 (中性)** | `Anamorphic` | 宽屏 | - | 指非方形像素的视频，播放时会自动拉伸至正确的宽高比，常见于DVD资源。 |

## 4. 片源来源
BluRay代表片源来自于蓝光原盘，常见来源如下：

#### <a id="uhd-blu-ray--blu-ray--bd--bdmv--iso"></a>UHD Blu-ray / Blu-ray / BD / BDMV / ISO
**画质源头，质量最高**。指未经任何修改的蓝光或4K UHD蓝光原盘完整内容或镜像文件。体积最大，包含所有花絮、菜单和多语言音轨。

#### <a id="remux"></a>REMUX
**无损提取，视听体验等同原盘**。直接从蓝光光碟而来，REMUX的意思是无损的提取出原光盘里面的视频数据、音频数据，封装到我们熟悉的视频封装格式中，适合收藏。普通观影个人感觉没必要。带`DIY`字样的为对蓝光盘再加工，汉化菜单、添加中文字幕、中文音轨等方便非英文网友观看。

#### <a id="web-dl"></a>WEB-DL
**官方数字源，画质极佳**。越来越流行的片源，质量第三，但是体积小了很多。全称为 web-download，是从官方发行的网络版直接搞下来的 （而不是屏幕录像） 。有些时候会加上具体的来源，例如 `NF-WEB-DL` 就是来自于 Netflix。

#### <a id="bdrip--brrip"></a>BDRip / BRRip
**有损压制，性价比之选**。Blu-ray Disc Rip，BD 转制。同样来源于蓝光光盘，只不过进行了重新编码，采用 [H.264](#h264--avc) 甚至 [H.265](#h265--hevc) 编码获得优秀的压缩比。质量略低于 [BD](#uhd-blu-ray--blu-ray--bd--bdmv--iso) 但文件较小，性价比很高。

#### <a id="hd-dvd--hd-dvdrip"></a>HD-DVD / HD-DVDRip
**历史的眼泪**。HD DVD（或称：High Definition DVD）是一种数字光储存格式的蓝色光束光碟产品，与蓝光光碟（Blu-ray Disc）相似且在2008年以前互为竞争关系，2008年厂商不再支持，这场持续了数年的规格之争，最终以蓝光的胜利而告终。所以，HD-DVD已成历史名词，目前网上流传带有HDDVD的片源，基本是2009年以前发行的。

#### <a id="hdtv--hdtvrip--hdrip"></a>HDTV / HDTVrip / HDrip
**电视录制，质量浮动**。常见于老电影。当时还没有 BD，流媒体也不流行。是数字采集 HD 电视得来的。在那个年代已经是最好的了。因为是电视，有可能会有台标。`HDTVrip` / `HDrip` 类似 `BDRip`，来源还是 HD 电视的采集，但是重新编码了。同类类似的`TVrip`代表电视采集转录（但不一定为HD电视的采集）。

#### <a id="webrip"></a>WEBRip
**录屏所得，质量有损**。从官方发行的网络版屏幕录像或者重新编码转制得来，质量可能相对于[WEB-DL](#web-dl)稍差。

#### <a id="hr-hdtv"></a>HR-HDTV
HR-HDTV准确名字叫做Half Resolution High Definition Television=HRHD（HR or HR-HDTV），也就是大家所说的全高清的一半，所以HR-HDTV也叫半高清。这个概念被提出来后，主要随着人人影视（YYTs）的壮大而推广，被大家拟熟知。

#### <a id="dvd--isodvd"></a>DVD / ISO(DVD)
**标清时代的最高标准**。指DVD原盘或其镜像文件。

#### <a id="dvdrip"></a>DVDRip
**DVD转制**。DVDRIP是从最终版的DVD转制，将DVD的视频、音频、字幕剥离出来，再经过压缩或者其他处理，然后重新合成成多媒体文件。一般来说，DVDrip由影音文件(后缀为avi)和字幕文件组成。

#### <a id="dvdscr--scr"></a>DVDScr / SCR
**预览版**。预览版的或者是测试版的DVD，非正式出版的版本。从预览版 DVD 中获取。比[DVDRip](#dvdrip)早发布，但画质稍差。(经常有一些不在黑边里在屏幕下方滚动的消息，包含版权和反盗版电话号码 ，会影响观看。)在欧美年末颁奖季时，给评委送审的片子可能就是这种片源。

#### <a id="r5"></a>R5
**特定区域早期版**。5区R5一般大都是俄罗斯5区版，由于北美大片在俄罗斯发行的语种经常配音为俄罗斯语，因此网络上会出现采用R5的视频+枪版[TS](#ts--telesync)的英语原声音频=合成版本(所以音效差)。但是，R5版不只是音效差，绝大部分情况视频质量也比欧美版DVD也差。R5区的DVD发布一般较欧美版早。

#### <a id="tc--telecine"></a>TC (Telecine)
**胶片版**。TC使用电视电影机从电影胶片直接数字拷贝。画面质量还不错,但亮度不足，有些昏暗。很多时候制作TC使用的音源来自[TS](#ts--telesync)，因此音质很差，但画面质量远好过TS。

#### <a id="ts--telesync"></a>TS (Telesync)
**盗录版**。TS与[CAM](#cam--camera)版的标准是相同的。 但它使用的是外置音源(一般是影院座椅上为听力不好的人设的耳机孔)。TS是在空的影院或是用专业摄像机在投影室录制，所以图象质量可能比CAM好。但画面的起伏很大。

#### <a id="cam--camera"></a>CAM (Camera)
**盗录版（标准“枪版”）**。CAM通常是用数码摄像机从电影院盗录。有时会使用小三角架，但大多数时候不可能使用，所以摄像机会抖动。因此我们看到画面通常偏暗、人物常常会失真。由于声音是电影院现场录制，所以经常会录到观众的笑声等杂音。因为这些因素，图象、声音质量通常都很差。

### 常见平台标识及其说明
文件名中经常会包含特定流媒体平台的缩写，以表明[WEB-DL](#web-dl)或[WEBRip](#webrip)的精确来源。
* **NF / Netflix (奈飞, 网飞)**: 全球流媒体巨头，以其丰富的原创内容和高码率的4K HDR视频流而闻名。
* **AMZN / Amazon / AZ (亚马逊)**: 亚马逊Prime Video，另一全球巨头，提供大量高质量4K HDR内容，是蓝光发售前的重要片源。
* **DSNP / Disney+ (迪士尼+)**: 迪士尼旗下平台，独家提供迪士尼、漫威、星球大战、国家地理等内容，部分影片提供画幅更高的IMAX Enhanced版本。
* **MAX / HMAX / HBO**: 华纳兄弟探索旗下平台，是HBO剧集、华纳电影的独家线上首发平台。
* **ATVP / Apple TV+ (苹果TV+)**: 苹果流媒体服务，以其极高的视频码率而著称，通常被认为是流媒体中画质的天花板。
* **iT / iTunes**: 苹果的数字内容商店，是购买和租赁数字电影的传统高质量来源，其WEB-DL资源一直享有盛誉。
* **B-Global / B-Intl / Bilibili (哔哩哔哩, B站)**: 中国领先的视频社区，尤其以动漫番剧和亚洲内容闻名。`B-Global`或`B-Intl`通常指其为海外用户提供的高清版本。
* **iQ / iQIYI (爱奇艺)**: 中国头部的综合性流媒体平台之一，提供大量剧集、综艺和电影。
* **Tencent / WeTV (腾讯视频)**: 中国另一大流媒体巨-头，拥有庞大的内容库。
* **Hulu**: 主要面向美国市场的流媒体服务，是许多美剧和电视节目的重要来源。

## 5. 视频编码
常见的编码格式如下：

#### <a id="av1"></a>AV1
**未来的标准，开放且免版税**。由开放媒体联盟（AOMedia）开发，目标是取代其前身[VP9](#vp9)，该联盟由半导体企业、视频点播供应商和网页浏览器开发商于2015年成立。测试结果显示AV1资料压缩率比[x264](#h264--avc)和[libvpx-vp9](#vp9)有大幅提升。

#### <a id="h265--hevc"></a>H.265 / HEVC (x265编码器)
**现在的主流，4K时代的核心**。是一种较新的编码格式。x265是一个开源项目，是一个将视频编码为H.265/高效率的视频编码（HEVC）格式的免费的库。H.265理论上会比H.264更节省空间，理论上能减小一半体积，但编码时间长点。并且，X265编码的视频对播放器解码能力要求也高。

#### <a id="h264--avc"></a>H.264 / AVC (x264编码器)
**曾经的王者，兼容性之王**。由国际电信联盟（iTU-T）所制定的新一代的视频压缩格式。H.264的数据压缩比能比当前DVD系统中使用的[MPEG-2](#mpeg-2)高2～3倍，比[MPEG-4](#mpeg-4)高1.5～2倍。

#### <a id="prores"></a>ProRes
**专业剪辑编码**。Apple ProRes 是苹果公司开发的高品质的有损影片压缩格式，可用于高达8K的后期制作。被广泛用于广告、纪录片、蓝光和流媒体中高清广播文件的最终交付编码。

#### <a id="vp9"></a>VP9
**Google主导的开放格式**。Google为了替换老旧的[VP8](#vp8)影像编码格式并与MPEG主导的[H.265/HEVC](#h265--hevc)竞争所开发的免费、开源的影像编码格式。VP9主要被用于YouTube。

#### <a id="vp8"></a>VP8
**VP9的前身**。是一个开放的格式，由Google主导以替代VP7，与之搭配WebM为一个容器格式，搭配声音格式则是使用Vorbis。

#### <a id="mpeg-4"></a>MPEG-4 (XviD/DivX)
**H.264前的网络霸主**。最显著特点就是 压缩率高且成像清晰，一般来说，一小时的影像可以被压缩为350M左右的数据。

#### <a id="vc-1"></a>VC-1
**微软主导的蓝光编码**。从压缩比上来看，[H.264](#h264--avc)的压缩比率更高一些，但是VC-1 格式的视频在解码计算方面则更小一些。

#### <a id="mpeg-2"></a>MPEG-2
**DVD时代标准**。MPEG-2特别适用于广播级的数字电视的编码和传送，被认定为SDTV和HDTV的编码标准。DVD影碟就是采用MPEG-2压缩标准。

## 6. 视频色深
10bit，常见的还有 8bit。代表像素点色彩空间使用几个bit表示。
我们知道计算机中所有颜色都是由红绿蓝（RGB）混合而成。每一种颜色称为一个通道。色深就是每个通道用多少空间去表示。所以，色深越大颜色表达的越精细，渐变也就越平滑。
8bit 是最常见的。三个通道就是 24bit，即所谓的「24 位真彩色」，一共约能表示出 1670 万种颜色。如果是 10bit，就能组合出 10.7 亿色。
不过不是所有显示器都支持 10bit 显示的，如果不支持，即使是 10bit 的视频，观感也和 8bit 一样。
另外，虽然直观上 10bit 版本的视频文件会更大。但实际上它们往往有更好的压缩率，所以有可能 10bit 的电影反而比 8bit 的要小。

## 7. 色彩动态范围
HDR 与SDR相对，简而言之画面高光亮处更亮，暗处更暗，画面对比动态范围更宽；因为自然界的太阳光强度最亮和最暗处光强动态范围极大，远大于屏幕二极管灯光亮度控制范围，因此HDR高光更亮更贴近自然界人眼视觉，画面更生动真实。等常见HDR标准有：

#### <a id="dolby-vision--dv--dovi"></a>Dolby Vision / DV / DoVi
由杜比实验室推出，与 [HDR10](#hdr--hdr10) 不同 Dolby Vision 采用了“动态元数据”技术，使得每一帧都获得了比较准确的最高和最低亮度，效果惊艳。常见复合标签：`DV.HDR10` 或 `Hybrid`，表示同时兼容DV和HDR10播放，兼容性最好。

#### <a id="hdr10--hdr10plus"></a>HDR10+ / HDR10PLUS
由三星、亚马逊、松下和20世纪福斯等公司联合推出的，可兼容 [HDR10](#hdr--hdr10)，具有与[杜比视界](#dolby-vision--dv--dovi)相似的优势，但不收取版权费用。添加了“动态元数据”技术。

#### <a id="hdr-vivid"></a>HDR Vivid
由 UWA 世界超高清视频产业联盟（原 CUVA ），以华为海思的技术为主导，推出的一个中国独创的 HDR 新标准，支持动态元数据，已成为国家广电行业的标准。

#### <a id="hdr--hdr10"></a>HDR / HDR10
由美国消费者技术协会推出，开源、免费的基础HDR标准。所有的 HDR 设备都支持，也是目前应用支持最广泛的 HDR 标准。但是 HDR10 采用的“静态元数据”技术。

#### <a id="hlg"></a>HLG (Hybrid Log Gamma)
由英国 BBC 和日本 NHK 联合发起，特点是没有元数据，可以同时兼容 HDR 显示屏和目前更为普及的 [SDR](#sdr) 显示屏，且兼容性特别好，并且是一个完全免费的标准。

#### <a id="sdr"></a>SDR (Standard Dynamic Range)
**标准动态范围**。这是所有非HDR视频的统称，也是我们过去几十年里观看所有视频（如DVD、普通蓝光、电视节目）的标准。它的亮度和色彩范围有限，是衡量以上所有HDR格式效果的基-准线。

## 8. 音频格式
`DDP5.1`是声音格式，它一般有两部分，前者`DDP`为音频编码，后者`5.1`代表声道形式：5方向声道加1个重低音声道，适配5.1功放音响系统，5.1系统也可写作6CH；同理7.1系统也可写作8CH；2.0系统（只有左右声道）也可写作2CH；CH是英文单词channel的缩写，代表一共有几个声道。常见声音编码有：

#### **第一梯队：无损沉浸式音频 (Lossless Immersive Audio)**
* #### <a id="dolby-atmos"></a>Dolby Atmos (杜比全景声)
  与传统的5.1、7.1基于声道的声音解决方案不同，杜比全景声是一种基于声音对象的3D环绕声技术，能够营造出动态的声音效果，是目前家庭影院的顶级标准之一。其内核通常是[Dolby TrueHD](#dolby-truehd)。
* #### <a id="dtsx"></a>DTS:X
  与[杜比全景声](#dolby-atmos)竞争的基于声音对象的沉浸式音频技术，对音箱布局要求更灵活。其内核通常是[DTS-HD MA](#dts-hd-ma)。

#### **第二梯队：无损多声道音频 (Lossless Channel-based Audio)**
* #### <a id="dts-hd-ma"></a>DTS-HD MA (Master Audio)
  **无损压缩**格式，与[Dolby TrueHD](#dolby-truehd)相似，是DTS的顶级格式。
* #### <a id="dolby-truehd"></a>Dolby TrueHD
  **无损压缩**编解码技术, 解码后与录音棚母版每一相应比特都一样。
* #### <a id="pcm--lpcm"></a>PCM / LPCM
  PCM声轨是录音棚母版的严格复制品，不用任何压缩地把信号编码到光盘上，保留了原材料的纯净性。
* #### <a id="flac"></a>FLAC
  与[MP3](#mp3)不同，FLAC是无损压缩，也就是说音频以FLAC编码压缩后不会丢失任何信息。
* #### <a id="ape"></a>APE
  一种无损压缩音频格式，通过Monkey”s Audio这个软件进行“瘦身”压缩为APE。

#### **第三梯队：高清有损多声道音频 (HD Lossy Audio)**
* #### <a id="dts-hd-hr"></a>DTS-HD HR (High Resolution Audio)
  与[杜比数字+](#dolby-digital--dd--ddp--eac3)相似，DTS-HD HR提供比标准[DTS](#dts)更高的比特率和更好的压缩改进。
* #### <a id="dolby-digital--dd--ddp--eac3"></a>Dolby Digital+ / DD+ / DDP / EAC3 (早期曾译作杜比数字增强)
  标准[杜比数字](#dolby-digital--dd--ac3)的增强版本，它提供更高的比特率和效率更高的压缩，声音质量也再一次得到提升。
* #### <a id="dts"></a>DTS
  [AC3](#dolby-digital--dd--ac3)的主要竞争对手，通常码率更高，理论上音质更好。
* #### <a id="dolby-digital--dd--ac3"></a>Dolby Digital / DD / AC3
  这个音频格式从DVD开始就为人们所熟知，几乎成了环绕声的代名词。

#### **第四梯队：通用有损音频 (Standard Lossy Audio)**
* #### <a id="opus"></a>Opus
  一个有损音频压缩的数字音频编码格式，由Xiph.Org基金会开发，目标是希望用单一格式包含声音和语音，适用于网络上低延迟的即时声音传输。
* #### <a id="aac"></a>AAC (Advanced Audio Coding)
  一种专为声音数据设计的文件压缩格式，与[MP3](#mp3)不同，它采用了全新的算法进行编码，更加高效，具有更高的“性价比”。
* #### <a id="ogg--vorbis"></a>OGG / Vorbis
  一种新的音频压缩格式，类似于[MP3](#mp3)等现有的音乐格式。但有一点不同的是，它是完全免费、开放和没有专利限制的。
* #### <a id="wma"></a>WMA
  WMA的全称是Windows Media Audio，是微软力推的一种音频格式，其压缩率一般可以达到1:18。
* #### <a id="mp3"></a>MP3
  这种压缩方式的全称叫MPEG Audio Layer3，能够在音质丢失很小的情况下把文件压缩到更小的程度。
* #### <a id="atrac"></a>ATRAC
  ATRAC压缩技术主要是利用了人耳的蔽遮效应，由于近年來编码压缩技术越來越成熟，所以经过编码解码过程后的声音仍直逼CD。

* #### <a id="wav"></a>WAV
  WAV是录音时用的标准的windows文件格式，文件的扩展名为“.wav”，无损但未压缩，体积巨大。

## 11. 帧率
电影最常见的帧率是24 帧/秒（电影产业标准）和30 帧/秒。随着高刷新显示装置的流行，一些压制组压制了60FPS或120FPS后缀的视频种子文件，码率相对更高。

## 12. 封装容器/后缀
常见视频文件的容器/后缀:

#### <a id="mkv"></a>mkv
后起之秀，MKV不是一种压缩格式，而是一种新的多媒体封装格式，也称多媒体容器 (Multimedia Container)。它可将多种不同编码的视频及16条以上不同格式的音频和不同语言的字幕流封装到一个Matroska Media文件当中。

#### <a id="mp4"></a>mp4
手机、平板的新宠儿，有的电影是MP4格式的，这代表着它采用了MPEG-4视频压缩编码。一般是用在掌上便携设备上（IOS/安卓平台，如手机、平板）播放。

#### <a id="ts--m2ts"></a>ts / m2ts
这个TS代表的是文件后缀名为.TS的高清格式。这种格式的优点在于对硬件的要求不高，缺点同样明显——超大的文件体积。`m2ts`是蓝光光盘上的视频文件格式。

#### <a id="mov"></a>mov
高清预告片首选，它是Apple公司开发的一种音频、视频文件格式，因具有跨平台、存储空间要求小等技术特点，得到业界的广泛认可。

#### <a id="webm"></a>webm
WebM 是一种开放的多媒体封装格式，通常与一些特定的视频和音频编码格式一起使用。WebM 最常用的视频编码格式是 [VP8](#vp8) 和 [VP9](#vp9)，而音频编码格式通常使用 Vorbis 或 Opus。

#### <a id="flv"></a>flv
(Flash Video) 是Adobe 公司推出的一种流媒体格式，由于其封装后的音视频文件体积小、封装简单等特点，非常适合于互联网上使用。

#### <a id="avi"></a>avi
廉颇老矣，AVI英文全称为Audio Video Interleaved，即音频视频交错格式。是将语音和影像同步组合在一起的文件格式。新片基本很少有人压avi格式了。

#### <a id="rmvb"></a>rmvb
RMVB由于很多年没有更新编码，再加上当初开发RMVB是为了做网络流媒体的，所以使用RMVB来压制高清是没有意义的。按理来说RMVB是早就应该淘汰的格式了。

## 13. 音轨语言信息
Dual.Audio/Dual-Audio: 双语音轨。
Tri-Audio: 三语音轨。
Multi: 多语种音轨。
CHINESE/RUSSIAN/KOREAN/HINDI/FRENCH: 使用语言名称直接标记出片源所包含的非英语单音轨。
DUBBED: 片源包含译制音轨，例如原电影为西班牙语电影，有人译制成英语并且配音加入到片源音轨当中。
Commentary: **评论音轨**，由导演或演员解说电影的特殊音轨。

## 14. 字幕信息
### 14.1. 内挂/内封字幕
封装到了资源文件中，但可通过播放器选择切换不同的字幕、隐藏字幕、调整字幕字体、时间轴等，包括SRT、ASS、PGS、MOV_TEXT等格式。

### 14.2. 外挂字幕
字幕网站下载到的字幕文件，如sub，ass，srt，拖到播放器中就能使用，与内挂类似。
外挂字幕一般有以下几种格式：
SRT: 最基本最流行的纯文本字幕。
SSA: 可以设置文字大小/效果，甚至添加水印。
ASS: SSA 增强版。
SUP: 图像版外挂字幕。
SUB与IDX: **DVD图形字幕**。这是从DVD中提取的图形字幕格式，由两个文件组成：`.sub`文件包含字幕的图片，`.idx`文件则包含了这些图片出现的时间码和位置信息。
外挂字幕也有一些特定后缀格式，例如：
语言后缀: .zh.srt代表中文字幕，.en.srt代表英文字幕。
优先级/数量后缀: .zh.2.ass代表第二个(优先级PLEX可识别)中文字幕。
为听力障碍者准备的字幕后缀: .hi代表Hearing-Impaired，.cc代表Close Captioned，.sdh代表Subtitles for the Deaf or Hard-of-Hearing。
中文字幕分为中文简体和中文繁体字幕（港澳台），一些地方和字幕组以.chs代表中文简体字幕，以.cht代表中文繁体字幕，例如Xx.chs.srt和Xx.cht.srt。

### 14.3. 硬字幕
又称内嵌字幕，就是把字幕跟画面一起压成了视频了，像水印一样，无法隐藏。
对于带硬字幕的片源，种子一般有以下字眼：
SUBBED: 画面内有不能消去的硬字幕。
UNSUBBED:没有内嵌字幕 与 SUBBED 对比。

---
## 15. 剧集/季信息
电视节目种子分为两类，分别是整季（Season Pack）打包和单集种子。
前者一般包含Season 01/S01/S1等字眼表示第一季（例）打包。
后者包含S01E02等字眼表示第一季的第二集单集（例），有的也可包含多集例如S01E01-E03。
注意一些动漫类的种子没有“季”的概念，只有剧集编号。
一些综艺节目的剧集编号以播出日期替代（少见）。

---
## 16. 种子大小信息
一些发布组例如GalaxyRG265会在种子标题包含视频文件大小信息，例如800M代表影视文件压制大小在800M左右，以此类推。

## 17. 其它杂项
* Anniversary.Edition: 周年纪念版。
* Bad A/R/Bad Aspt: 画面比例不正常。
* Bad IVTC: 帧率错误，画面有不可去除的拉丝。
* CEE/EUR/US/HK/etc: 指蓝光光盘的发行区域。
* Complete: 完整版，常用于剧集整季打包，表示该季已完结。
* Director.Cut/DC: 导演剪辑版。
* DIY: 指发布者在原盘或他人发布资源的基础上，自己动手添加了其他来源的音轨或字幕。
* DivX: 一种早期的MPEG-4视频编码格式。
* DVDR: 由DVD直接复制而来，保留原始MPEG2视频编码的版本。
* Extended: 加长版。
* FS: Full Screen: 4:3 全屏幕版本的影片。
* Hybrid: 混合版，指将不同来源的视频、音频或字幕素材混合制作的版本。
* IMAX: 巨幕版，通常画面比例更高(1.90:1或1.43:1)，能显示更多内容。
* INTERLACED: 隔行扫描问题，画面有拉丝，是压制错误的表现。
* Internal / iNTERNAL: 内部版，指PT站点的压制组为本站首发或测试而发布的版本。
* LIMITED: 限制上映版，指在少数影院上映的电影。
* NUKED: 被废弃的版本，因画质太差或其他严重问题，被站点规则判定为无效种子。
* Open.Matte: 满屏版，与宽屏版相比，通常上下黑边更少。
* PROPER: 完美版，表明此版本优于之前其他小组发布的有问题的版本。
* R-RATE: 有删节。
* RECODE: 指对另一小组发布的版本进行修正或重新编码的版本。
* REMASTERED/4K: 重新灌录版/4K修复版。
* REPACK: 修正版本，发布组对自己之前发布的有问题的版本进行修正后重新发布。
* SVCD: 一种早于DVD的视频光盘格式。
* Theatrical.Cut: 剧场版（影院上映版本）。
* UNRATE: 未删节版。
* WITH.EXTRAS: 带花絮。
* WITH.OST: 带原声音乐CD。
* WP/WORKPRINT: 工作录影带版，电影制作过程中的版本，非最终版。
* WS: Wide Screen: 16:9或更宽的宽屏幕版本的影片。

    * **视频编码**: `HEVC` (即[H.265](#h265--hevc))
    * **发布组**: `GalaxyTV`
