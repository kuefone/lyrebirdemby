#### 1.1 ❓ 为什么我的影片播放时有些卡顿？ {#q-playback-lag}

:::info 💡 解决方案概要
琴鸟现在采用了 **硬盘 (服务器在德国) + `115`/`Google Drive` 网盘** 的混合存储模式。无论您访问哪里的资源，卡顿 99% 都与网络连接质量有关。

1.  访问 **硬盘 (德国)** 上的资源，您与服务器的物理距离很远。
2.  访问 **`Google Drive`** 或 **`115`** 上的资源，会受到网络服务商（ISP）或防火墙的国际出口限制。

这并非无法解决。我们为您提供了两种主流的解决方案，您可以根据自己的情况选择。
:::

#### 1.1.2 ✅ 方案二：搭建自己的反向代理 (进阶玩法) {#q-playback-lag-reverse-proxy}

:::tip 方案二：✅ 搭建自己的反向代理 (进阶玩法)
这是一个为喜欢折腾、动手能力强且拥有自己海外服务器（`VPS`）的朋友准备的终极方案。它能为您打造一条完全私有、稳定且高效的“网络超跑赛道”。

**核心理念**：琴鸟提供一个稳定的服务器IP地址（裸IP线路），您可以利用自己位于海外的 `VPS`，通过我们提供的一键脚本，将您的域名反向代理到琴鸟的服务器IP上。从此，您只需访问自己的域名，就能以最佳线路高速访问Emby。
:::

##### **您需要准备什么？**

1.  一台位于海外的**`VPS`服务器** (推荐使用 `Debian 11` 或 `Ubuntu 22.04` 以上的系统)。
2.  一个您自己拥有的**域名**。
3.  从 [**账号管理机器人**](https://t.me/Lyrebird_bot) 获取服务器的**裸IP线路地址**。

##### **什么是反向代理 (`Reverse Proxy`)？**

简单来说，反向代理 (`Reverse Proxy`) 就是在您的 `VPS` 上架设一个“中转站”（通常使用 `Nginx` 或 `Caddy`）。

* **数据流**: `您的设备` -> `您的域名` (解析到您的VPS) -> `您的VPS` (中转) -> `琴鸟的裸IP`
* **好处**:
    1.  **独享带宽**: 您不再需要和别人挤 `Cloudflare` 的公用通道。
    2.  **路径优化**: 您可以选择到琴鸟服务器延迟最低的 `VPS` 位置。
    3.  **安全性**: 您可以通过自己的域名和 `HTTPS` 访问，进一步增强隐私。
    4.  **稳定性**: 避免 `Cloudflare` 节点波动带来的影响。

##### **推荐的 SSH 工具**

您需要 `SSH` 客户端来登录您的 `VPS` 执行脚本。

* **Windows**: 推荐使用 `Xshell` 或 `MobaXterm`。
* **macOS / Linux**: 系统自带的 `Terminal` 或 `iTerm2`。
* **全平台**: `Termius`。

:::tip 教程哪里找？
这些都是非常成熟的工具。如果您不熟悉如何使用 `SSH` 登录服务器，建议前往 `Bilibili` 或 `YouTube` 搜索“`Xshell 使用教程`”或“`Termius 入门`”，有海量的视频手把手教学。
:::

##### **一键反代脚本使用指南**

:::danger ⚠️ 操作警告
此操作需要您具备基础的 `SSH` 和服务器操作知识。请确保您知道自己在做什么。
:::

1.  **登录您的VPS**
    * 通过 `SSH` 工具登录到您的海外服务器。

2.  **执行一键脚本**
    * 复制以下整行命令，粘贴到您的服务器命令行中，然后按下回车。（大多数终端和GitHub页面都会在此代码块右上角提供一个复制按钮）
    > **👉 一键执行命令：**
    ```bash
    sudo bash -c "$(curl -fsSL [https://raw.githubusercontent.com/kuefone/forwardlyrebirdemby/main/fdlyrebird.sh](https://raw.githubusercontent.com/kuefone/forwardlyrebirdemby/main/fdlyrebird.sh))"
    ```

3.  **按照提示操作**
    * 脚本执行后，会以交互方式引导您：
        * 首先，它会要求您输入**您自己的域名**（例如 `emby.yourdomain.com`）。
        * 接着，它会要求您输入**琴鸟的服务器裸IP地址**。
    * 输入完毕后，脚本会自动为您完成安装 `Nginx`、配置反向代理、申请 `SSL` 证书等所有繁琐的工作。

4.  **完成与使用**
    * 脚本成功运行结束后，您就可以通过浏览器访问 `https://emby.yourdomain.com` 来登录您的Emby账号了。
    * 之后，您可以参照官网教程，将您所有设备的Emby服务器地址都换成您自己的域名。

您已经拥有了一条专属于您的、点对点的私人观影高速公路。

#### 1.3 ❓ 琴鸟的混合存储（硬盘+网盘）有什么区别？我该如何选择？ {#q-hdd-vs-cloud}

:::info 💡 核心区别
我们选择混合模式，是为了给您“极致体验”和“海量片库”的双重选择，两者兼得！

您可以通过文件名中是否包含 `` `[115]` `` 或 `` `[GD]` `` 标签来轻松区分它们。
:::

| 特性 | 🌍 硬盘 (服务器在德国) | ☁️ 115/GD 网盘 (我们的海量片库) |
| :--- | :--- | :--- |
| **片库容量** | **有限** (当前 `88TB`) | **近乎无限** (`5PB` + `♾️`) |
| **资源定位** | 存放**热门**、**高频**、**高码率**的精品资源 | 存放**全量**、**冷门**、**海量**的剧集和电影 |
| **加载速度** | ⚡ **如丝般顺滑**，几乎无需缓冲，进度条可随意拖拽 | 🐢 速度依赖您的网络，拖拽可能需缓冲 |
| **智能功能** | ✅ **支持**扫描并**一键跳过片头/片尾** | ❌ **不支持** |
| **数据安全** | 🛡️ **固若金汤**，资源由我们自己保管 | 🛡️ **相对安全**，但理论上受服务商政策影响 |
| **入库速度** | 需下载时间，相对较慢 | 极快（115秒传/GD转存），求片几乎秒到 |

:::tip 💡 总结
* 追求**最佳观影体验**（如丝滑拖拽、跳过片头），请优先选择**不带网盘标签**的物理服资源。
* 追求**海量片库**、**快速求片**，网盘资源是您的坚实后盾。

## 🇨🇳 1.4 白名单线路 (大陆优化) {#whitelist-line}

:::info 💡 **一句话总结**
一条中国大陆优化直连线路，**所有用户均可使用**，但非白名单用户会计费。
:::

* **核心特点**：针对中国大陆网络环境进行三网优化，在不使用代理工具的情况下，可获得比主线路更好的直连效果。
* **使用成本**：
    * **白名单用户**：免费使用。
    * **非白名单用户**：**可以使用**，但系统会自动从您的账户中扣除 **`10鸟币/天`** 的服务费（按自然日计算，当天使用即扣费）。

:::tip 速率限制说明
为了保证线路质量和公平性，此线路对所有用户统一限速 **`100Mbps`**。
:::

::: details 🤔 在部分地区无法连接？点击查看域名解析IP直连教程 {#whitelist-line-dns}

---

部分地区（如江苏等）的网络环境可能存在“墙中墙”，直接屏蔽了未备案的域名。别担心，您可以通过手动解析IP的方式绕过这个限制。

#### 1.4.1 获取IP地址 {#whitelist-line-get-ip}

1.  打开DNS查询网站，例如 [**站长工具DNS查询**](https://tool.chinaz.com/dns/)。
2.  在输入框中，输入 **白名单线路的域名**，点击“检测”。
3.  在检测结果中，复制任意一个解析出的 `IP` 地址。

    ![DNS解析](https://raw.githubusercontent.com/kuefone/wiki/main/docs/images/whitelinedns.png)
    ![IP解析](https://raw.githubusercontent.com/kuefone/wiki/main/docs/images/whitelineip.png)

#### 1.4.2 修改客户端配置 {#whitelist-line-set-ip}

1.  在您的 Emby 客户端中，将服务器地址栏中的 **域名部分**，替换为您刚刚复制的 **`IP` 地址** 即可。
2.  **端口号保持不变**。

:::danger 重要：IP地址并非永久固定！
请注意，解析出的 `IP` 地址大约 **每周会变动一次**。当您发现无法连接时，只需重复上述步骤，获取最新的 `IP` 地址即可。这是在特殊网络环境下保持连接的必要操作。同时，请爱护这条脆弱的优化线路，**切勿泄露解析出的 IP 地址**。
:::

## 📥 3. 求片与资源入库 {#faq-requests}

<a id="q-import-delay"></a>
#### 3.1 ❓ 求片成功了，但媒体库里迟迟没有出现？ {#q-import-delay}

:::info 💡 请给机器人一点处理时间
**它正在后台努力工作中。**

您可以把这个过程想象成一个包裹的物流：
1.  **下单成功**：机器人提示您“下载完成”（或“转存成功”）。
2.  **仓库分拣**：系统自动将文件整理、归类到正确的货架（媒体文件夹）。
3.  **系统录入**：`Emby` 服务器扫描货架，识别到新包裹后，录入系统。
4.  **上架展示**：影片信息和海报出现在您的媒体库，可以观看啦！
:::

:::tip 💡 入库时间
* 访问**硬盘 (德国)** 的资源，这个自动化流程通常在 **15分钟内** 就能走完。
* 访问**网盘**的资源（特别是 `115` 秒传），这个过程**通常会更快**。

如果超过半小时仍未出现，可能是某个环节遇到了小麻烦，您可以在群里友好地留言并 `@` 相关管理员，他们上线后会为您查看。
:::

<a id="q-scraping-fail"></a>
#### 3.2 ❓ 下载完成了，但影片为什么一直不入库？ {#q-scraping-fail}

:::info 💡 Emby 看不懂“火星文”
**这通常是因为影片的文件名像“火星文”，Emby的识别程序 (`Scraper`) 看不懂。**

* **“刮削”是什么？** `Emby` 会像一位图书管理员，通过读取文件名（书名），去网上数据库 (`TMDb` 等) 匹配并抓取影片的海报、简介、演员表等信息（录入图书卡片）。
* **为什么会失败？** 如果文件名非常混乱或信息错误，`Emby` 就无法识别它究竟是哪部作品，自然也就无法为它制作“图书卡片”并将其上架到媒体库中。
:::

:::tip 💡 解决方案
这种情况需要管理员手动为文件更名并校对信息，请您耐心等待管理员上线后处理。
:::

<a id="q-no-subtitles"></a>
#### 3.3 ❓ 我求的影片没有中文字幕怎么办？ {#q-no-subtitles}

:::info 💡 求片小技巧
在选择资源时，优先挑选文件名中带有字幕组或压制组标签的影片。

这些版本的影片通常已经为您精心内嵌了高质量的字幕，可以免去后续的烦恼。（当然 `Lyrebird` 下片机器人处也有 `AI机翻`，`5鸟币/次`，也可以对付一口）
:::

* **推荐的资源标签**：寻找文件名后缀带有 `ADweb`, `HHweb`, `Mweb`, `Duboku`, `ViEu` 等标签的资源。
* **如果已经求了无字幕版本，怎么办？**
    1.  您可以自行前往专业的字幕网站（如 `SubHD`、`字幕库` 等）寻找匹配的字幕文件（通常是 `.srt` 或 `.ass` 格式）。
    2.  下载后，在Telegram的下片机器人对话框中，发送 `/zimu` 命令，按照机器人的温柔提示上传您的字幕文件，它会自动为您心爱的影片配上字幕。

<a id="q-request-fail"></a>
#### 3.4 ❓ 为什么我求片失败了？ {#q-request-fail}

:::info 💡 常见原因
**这往往不是您的错，而是因为您想找的资源在网络世界中已经“绝版”了。**

最常见的原因是：
1.  您请求的资源在 `PT` (Private Tracker) 站点中已被删除，或者已经没有任何人分享它（做种）。机器人也无能为力，因为它找不到源头。
2.  `PT` 站点冷却中，过会再查找。
3.  种子被一些禁止下载的资源关键词屏蔽了。
4.  种子做种数小于 `10`，`qbittortent` 下载容易卡住，已被屏蔽。
5.  该资源的 `PT` 站点 `Lyrebird` 未曾拥有，可以问问神通广大的管理员。
:::

:::tip 查种机器人
`Lyrebird` 群内有查种机器人 `@pteebot`，您可以在聊天框内输入 `@pteebot 资源名称 S00 E00` （不需要发送内容，记得输入 `@pteebot` 后要点一下出来的机器人再输入后面的内容，记得加空格）等关键词查看 `PT` 是否有种子，有了再去下片机器人处下片。
:::

<a id="q-subscribe-delay"></a>
#### 3.5 ❓ 我订阅的剧集为什么没有自动更新？ {#q-subscribe-delay}

:::info 💡 并非实时，请耐心等待
**这是由订阅功能的运行机制决定的，它是一位勤勉但需要按时巡逻的哨兵。**

订阅功能依赖后台程序**周期性地**自动搜索全网资源，而不是实时监控。所以请您放心，新发布的剧集并没有被遗忘，它会在下一次系统巡逻时被发现、下载并送到您的媒体库中。
:::

<a id="q-missing-episodes"></a>
#### 3.6 ❓ 剧集播放列表里好像缺了几集，该怎么办？ {#q-missing-episodes}

:::info 💡 分情况讨论
**别担心，我们可以分情况轻松解决。**

* **如果只缺了零星的1-2集**：
    * 我们建议您尝试在 `Lyrebird` 下片机器人中进行精确搜索（例如 `` `老友记 S01E05` ``）来手动补全。这通常是最快的方式。
* **如果缺失了比较多的集数**：
    * 请您在群里发一条清晰的消息，说明“**哪部剧、哪一季、缺失了哪些集数**”，并 `@` 友好的管理员，他们上线后会非常乐意为您进行批量补齐。
:::

## 💾 5. 服务器与资源 {#faq-server}

<a id="q-server-capacity"></a>
#### 5.1 ❓ 鸟服的存储空间有多大？我收藏的影片会被删除吗？ {#q-server-capacity}

:::info 💡 空间充足，策略透明
**我们的存储空间是所有成员共享的宝贵资源，它很充足，并会与社区一同成长。**

* **当前容量**：琴鸟服务器当前总容量已达到：
    * 🌍 **硬盘 (德国)**：**`88TB`**
    * ☁️ **`115` 网盘**：**`5PB`**
    * ☁️ **`Google Drive`**：**`GD♾️` (无限容量)**
* **增长策略**：我们的资源库仍在持续、稳健地增长中，请您放心使用，空间还非常宽裕。
* **清理策略**：我们珍视每一部影片。**极少会删除**资源。只有在**物理硬盘**空间真的非常紧张时，我们才会考虑转移那些**长达数月甚至更久都无人问津**的超冷门资源，以便为更受欢迎的新内容腾出家园。（网盘资源近乎无限，基本不会清理）
